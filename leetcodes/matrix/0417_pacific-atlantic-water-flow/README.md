## 問題描述

有一個 m x n 的矩陣，每一個格子代表一座島嶼，四周被兩個海洋包圍：

- Pacific Ocean（上緣與左緣）
- Atlantic Ocean（下緣與右緣）

給予 m x n 的 heights matrix，代表每個格子（島嶼）的高度。規則如下：

1. 島嶼接收到雨水，雨水可以流向四個方向中高度**小於等於**當前格子的鄰居節點
2. 雨水可以從任何島嶼出發往四周流動

回傳所有座標 (coordinate)，使得該格子的雨水能**同時**順利流向 Pacific 與 Atlantic。

## Constraints

- m == heights.length
- n == heights[r].length
- 1 <= m, n <= 200
- 0 <= heights[r][c] <= 10^5

## 解題思路

### 初步分析（觀察與發想）

- 矩陣、島嶼、連通 → 直覺聯想到 BFS / DFS。
- 邊界是特殊的，題目要求雨水要能同時抵達兩個邊界 → 嘗試從邊界出發思考。
- 正向思考：從每個島嶼出發試著到達兩個邊界，路徑大量重複，複雜度約為 O((m×n)²)，不可行。
- **反向思考**：從邊界出發，找出哪些島嶼的雨水「可以回流到」該邊界（等效於水往高處逆流）。遍歷兩次後取交集，交集就是同時可以抵達兩個邊界的島嶼。

## 解法總覽

### Solution 1: BFS

- **核心策略**：

  反向思考，分別以 Pacific 邊界與 Atlantic 邊界為起點，各做一次 BFS。

  從邊界往內部「逆流」擴散——只走向高度**大於等於**當前節點的鄰居（反向後，水可以往高處爬）。

  兩次 BFS 各自收集「可以回到出發邊界」的節點集合，取交集即為答案。

- **思路說明**：

  **關鍵認知**：

  「反向流動」的限制體現在高度比較條件上，而非移動方向上。

  兩個 BFS 的探索方向都是四個方向（上下左右），差異只有**起點不同**。

  BFS 將鄰居節點加入 queue 的條件：

  1. 未越界
  2. 尚未拜訪過
  3. 鄰居高度 ≥ 當前高度（確保反向逆流合法）

  **Pacific BFS（從上緣 + 左緣出發）**：

  - 將整行左緣（col = 0）與整列上緣（row = 0）的所有節點加入初始 queue、visited、結果集
  - 邊界節點本身就是「可以回到 Pacific」的節點，必須直接收入結果集
  - BFS 向四個方向擴散，持續收集可以逆流回 Pacific 的節點

  **Atlantic BFS（從下緣 + 右緣出發）**：

  - 將整行右緣（col = n-1）與整列下緣（row = m-1）的所有節點加入初始 queue、visited、結果集
  - 同上，邊界節點直接收入結果集
  - BFS 向四個方向擴散，持續收集可以逆流回 Atlantic 的節點

  最後遍歷 Pacific 結果集，找出同時存在於 Atlantic 結果集的節點，轉換回 `[row, col]` 格式回傳。

  **易錯點**：邊界節點若只加入 visited 和 queue 而未加入結果集，角落節點（同時屬於兩個邊界）與 1×1 矩陣將無法出現在交集中。

- **複雜度分析**：

  - 時間複雜度：O(m × n)（每個節點最多被拜訪一次）

  - 空間複雜度：O(m × n)（visited set + queue 最壞情況涵蓋所有節點）

  - 通過狀態：✅ AC

- **測試案例**：

  - 案例 A（1×1 退化輸入）：

    Input: `[[5]]`

    Output: `[[0,0]]`

    Expected: `[[0,0]]`

    Explain: 唯一節點同時是四個邊界的一部分，應出現在答案中。驗證邊界節點直接收入結果集的邏輯。

  ***

  - 案例 B（單列矩陣）：

    Input: `[[1,2,3]]`

    Output: `[[0,0],[0,1],[0,2]]`

    Expected: `[[0,0],[0,1],[0,2]]`

    Explain: 只有一列時，row 0 同時是 Pacific 上緣與 Atlantic 下緣，所有節點皆屬於兩個邊界，全數應出現在答案中。

  ***

  - 案例 C（全部高度相同）：

    Input: `[[2,2],[2,2]]`

    Output: `[[0,0],[0,1],[1,0],[1,1]]`

    Expected: `[[0,0],[0,1],[1,0],[1,1]]`

    Explain: 高度相同時，等號條件（≥）應允許逆流，所有節點皆可達兩個邊界。驗證高度比較條件邊界值的正確性。

  ***

  - 案例 D（標準範例，驗證一般路徑）：

    Input: `[[1,2,2,3,5],[3,2,3,4,4],[2,4,5,3,1],[6,7,1,4,5],[5,1,1,2,4]]`

    Output: `[[0,4],[1,3],[1,4],[2,2],[3,0],[3,1],[4,0]]`

    Expected: `[[0,4],[1,3],[1,4],[2,2],[3,0],[3,1],[4,0]]`

    Explain: LeetCode 官方範例，驗證一般情況下交集邏輯與路徑擴散的正確性。

### Solution 2（DFS）

- **核心策略**：

  與 Solution 1 相同，反向從邊界出發，逆流往高處爬。差異在於將 BFS 的迭代展開改為遞迴的深度優先展開，每次進入一個節點後，立即遞迴處理其合法鄰居。

- **思路說明**：

  將 BFS helper 改寫為遞迴 DFS helper，進入函式時先將當前節點加入 reachable，再對四個方向的鄰居逐一判斷：

  1. 未越界
  2. 尚未在 reachable 中
  3. 鄰居高度 ≥ 當前高度（反向逆流條件）

  三個條件全部滿足才遞迴進入鄰居。起點初始化與取交集的方式與 Solution 1 完全相同。

- **複雜度分析**：

  - 時間複雜度：O(m × n)（每個節點最多被拜訪一次）
  - 空間複雜度：O(m × n)（reachable set + 遞迴呼叫堆疊，最壞情況下堆疊深度為 m × n）
  - 通過狀態：✅ AC

- **測試案例**：同 Solution 1 測試案例
