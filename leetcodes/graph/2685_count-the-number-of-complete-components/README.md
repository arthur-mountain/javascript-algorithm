## 問題描述

給定一個整數 n，以及代表邊的 2D array edges，表示一個含有 n 個節點（編號 0 ~ n - 1）的無向圖，請回傳「完全連通元件」的數量。

連通元件定義：圖中的一個子圖，其中任意兩個節點之間都存在一條路徑（可以透過中間節點間接連通，不一定要直接相連）；且此子圖內的任何節點與子圖外部的節點之間都沒有邊相連。滿足以上兩個條件，該子圖即為一個連通元件。

完全連通元件定義：若一個連通元件中的每一個節點，與該子圖內除自己以外的所有其他節點之間都存在一條邊（直接連線），則此連通元件被稱為「完全的」。

## Constraints

- 1 <= n <= 50
- 0 <= edges.length <= n \* (n - 1) / 2
- edges[i].length == 2
- 0 <= a<sub>i</sub>, b<sub>i</sub> <= n - 1
- a<sub>i</sub> != b<sub>i</sub>
- There are no repeated edges.

## 解題思路

### 初步分析(觀察與發想)

- 圖論問題，可用 DFS 或 BFS 求解。

- 需要先建立圖，因為節點編號是連續整數 0 ~ (n - 1)，適合用 Adjacency List，具備空間利用、遍歷與存取效率高的優勢。

- 曾嘗試「一次迴圈邊建立圖、邊判斷是否為『完全連通元件』」，後來發現不可行，原因如下：

  - 「連通性」判斷本身沒問題：其狀態具備單向合併性（動態連通，如 Disjoint Set），新節點的加入不會改變舊子圖已連通的事實。

  - 但「完全連通（Complete）」要求子圖內的每一個節點都必須與其他所有節點相連，這個條件在建圖過程中無法即時判斷：新加入的邊會產生「後效性」，回頭改變舊子圖的完全性狀態。因此，這類具備後效性的條件無法在變動狀態下即時結算，必須採取**「先完整建圖、後走訪處理」**的兩階段解法。

## 解法總覽

### Solution1：

- **核心策略**：

  先完整建圖、再走訪處理。

  初始化長度為 n 的 graph，避免動態擴充陣列。

  初始化 visited 陣列，用來判斷節點是否已被訪問過（即是否已被算進某個走訪過的連通元件中）。

  初始化 count 為 0，作為完全連通元件的計數。

  1. 先建立圖。
  2. 遍歷圖中的每個節點，使用 DFS 走訪，收集當前連通元件的所有節點。
  3. 遍歷該連通元件內的所有節點，判斷其出度（鄰居數）是否等於 k - 1
     （因為完全連通元件的特性，要求連通元件內的每一個節點都必須與其他所有節點相連）。
  4. 若上述檢查通過，代表此為完全連通元件，count + 1。

- **思路說明**：

  這題的核心觀察是：連通性與完全性是兩種不同「時間特性」的性質。連通性可以邊建圖邊判斷，但完全性有後效性——新加入的邊可能讓原本以為不完全的子圖後來變完全，因此無法在建圖過程中即時判斷。這也是核心策略採用「先建圖、後走訪」兩階段的原因。

  走訪工具選 DFS 是當下直覺，BFS 同樣可行；曾考慮過 Union-Find，但它的強項在於「動態」連通性，這題 edges 是一次給齊的靜態輸入，用不到，還得額外維護 parent/height 結構，所以沒選。

  核心策略裡「出度 = k - 1」這個判斷條件，是理解完「完全連通」定義後自然帶出來的：分量大小為 k 時，每個節點理論上要連到除了自己以外的 k - 1 個節點，出度自然等於 k - 1。

- **複雜度分析**：

  - 時間複雜度：O(V + E)

    - 看似有雙層迴圈與遞迴，但因為 visited 的紀錄，每個節點實際上只會被訪問一次，因此整體仍是線性掃描。

  - 空間複雜度：O(V + E)

  - 通過狀態：✅ AC

- **測試案例**：

  - 案例 A（極小輸入）:

    Input: n = 1, edges = []

    Ouput: 1

    Expected: 1

    Explain: n = 1 時圖中只有一個節點、沒有任何邊。依定義該節點自身即為一個連通元件，且出度 0 恰好等於 k - 1 = 1 - 1 = 0，滿足完全條件。這個案例驗證 k = 1 時「k - 1」公式是否仍成立、不會產生負值或誤判。

  ***

  - 案例 B（特殊結構：斷開的圖，完全與不完全並存）:

    Input: n = 6, edges = [[0,1],[0,2],[1,2],[3,4],[4,5]]

    Ouput: 1

    Expected: 1

    Explain: 圖中有兩個連通分量：{0,1,2} 形成三角形，每個節點出度都是 2 = k - 1，屬於完全連通元件；{3,4,5} 形成一條鏈（3-4-5），節點 3 和 5 的出度只有 1，未達到 k - 1 = 2，因此不完全。此案例專門驗證「是否逐分量獨立判斷完全性」，而非誤用全域出度或漏算某個分量。

  ***

  - 案例 C（極值：邊數達上限的完全圖）:

    Input: n = 4, edges = [[0,1],[0,2],[0,3],[1,2],[1,3],[2,3]]

    Ouput: 1

    Expected: 1

    Explain: 邊數達 constraints 上限 n \* (n - 1) / 2 = 6，整張圖本身就是唯一的連通分量，也是完全圖。驗證邊數達最大值時程式仍正確運行，且「整圖即單一完全分量」的判斷無誤。

  ***
