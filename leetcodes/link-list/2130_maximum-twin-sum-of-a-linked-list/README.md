## 問題描述

在一個長度為 n 的鏈結串列中（n 為偶數），第 i 個節點和第 (n-1-i) 個節點稱作孿生節點。

孿生和定義為孿生節點的值之和。請找出並回傳該鏈結串列中最大的孿生和。

## Constraints

- 鏈結串列長度為偶數，範圍在 2 到 100,000 之間。
- 節點數值範圍在 1 到 100,000 之間。

## 解題思路

### 初步分析

- 由於鏈結串列是單向的，無法直接取得尾部節點。若要計算對稱節點之和，必須解決回溯或反向存取的問題。
- 可以選擇使用額外空間 (Stack) 處理，或是原地修改指標 (反轉鏈結) 來達成空間優化。

## 解法總覽

### Solution 1：Stack

- **核心策略**：

  利用快慢指標與 Stack，在遍歷過程中將前半段節點放入 Stack。此時，慢指標剛好停在後半段起點，後續遍歷後半段時與 Stack 出棧節點配對計算。

- **思路說明**：

  1. 快指標每次走兩步，慢指標每次走一步。在慢指標移動的同時，將經過的節點依序推入 Stack。

  2. 當快指標到達終點時，慢指標剛好位於後半段的起點。

  3. 開始從慢指標位置往後遍歷，同時從 Stack 彈出節點，將兩者數值相加並更新最大值。

- **複雜度分析**：

  - 時間複雜度：O(n)

  - 空間複雜度：O(n)

---

### Solution 2：原地反轉 (兩趟遍歷)

- **核心策略**：

  利用快慢指標定位中點，並將**後半段**鏈結原地反轉。接著，同步移動頭指標與反轉後的後半段指標進行計算。

- **思路說明**：

  1. 使用快慢指標將鏈結分為兩半，並找到後半段的起點。

  2. 將後半段的所有節點進行原地反轉，改變鏈結方向。

  3. 使用兩個指標，分別指向鏈結頭節點與反轉後的後半段頭節點，同步向後移動並計算對稱節點之和。

- **複雜度分析**：
  - 時間複雜度：O(n)
  - 空間複雜度：O(1)

---

### Solution 3：單趟原地反轉 (極致優化)

- **核心策略**：

  相較於 Solution 2 反轉後半段，此解法直接原地反轉前半段。將找中點與反轉前半段合併為單次迴圈操作；

  遍歷結束後，慢指標剛好落在後半段起點，且前半段已完成反轉，此時兩指標恰好位於中點，接著向兩側移動計算孿生和。

- **思路說明**：

  1.  遍歷過程中，快指標走兩步，慢指標走一步。

  2.  關鍵在於「反轉動作」：在慢指標移動的同時，將當前節點指向已處理好的反轉串列，直接原地改變指標指向。

  3.  中點對稱：

      當迴圈結束時，前半段鏈結已經被反轉過來，且慢指標停在後半段起點。

      這兩組指標分別位於鏈結的中點兩側(前半段的終點、後半段的起點)，此時透過同步移動，實現從中間向兩側擴散的對稱計算。

- **複雜度分析**：

  - 時間複雜度：O(n)

  - 空間複雜度：O(1)

- **其他備註**：

  - 效能最優，減少了遍歷次數與迴圈開銷。

  - 使用初始空指標簡化了邊界判斷，讓邏輯更為流暢。
