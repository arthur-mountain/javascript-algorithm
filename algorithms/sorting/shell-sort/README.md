# Shell Sort（希爾排序）

先用大間距做粗略排序（讓遠距離的錯位元素快速歸位），再逐步縮小間距做精細排序。本質是「帶間距的 [Insertion Sort](../insertion-sort/README.md)」。

## 觸發條件

中等資料量、嵌入式系統（程式碼簡單 + in-place + 比 O(n²) 快，但不需要 O(n log n) 保證）。

## 運作流程（執行期）

1. 設定初始間距 `gap`。
2. 對每個 gap 做一次「帶間距的 Insertion Sort」（步長從 1 變為 gap）。
3. 縮小 gap，重複直到 gap = 1。
4. gap = 1 時就是標準 Insertion Sort，但此時陣列已大致有序，收斂很快。

## 逐步範例

`input = [8, 5, 3, 7, 6, 1, 2, 4]`（n = 8）。Knuth 序列取不超過 n/3 的最大 gap → 從 `gap = 4` 開始，再降到 1。

**gap = 4**：對「相距 4」的子序列各自做 Insertion Sort，把遠距離錯位元素快速拉近。

```
索引:   0  1  2  3  4  5  6  7
        8  5  3  7  6  1  2  4
子序列: (8,6)  (5,1)  (3,2)  (7,4)     ← 每組相距 4
排序後:  6,8    1,5    2,3    4,7
結果:   6  1  2  4  8  5  3  7
```

**gap = 1**：此時陣列 `[6,1,2,4,8,5,3,7]` 已大致有序（大值多半偏右），標準 Insertion Sort 只需少量移動即可收斂為 `[1,2,3,4,5,6,7,8]`。

**對比**：若一開始就 gap=1（純 Insertion Sort），把 8 從 index 0 移到末端要 7 次相鄰移動；gap=4 一步就把 8 往右挪了 4 格——這就是大間距「先粗排」省下的工作量。

## 關鍵洞察

Insertion Sort 的弱點是「每次只能移動一格」，逆序對多時很慢。Shell Sort 用大間距讓遠距離的錯位元素快速接近正確位置，大幅減少後續小間距排序的工作量。複雜度取決於 gap sequence 的選擇。

## Gap Sequence

| 名稱             | 序列             | 最差複雜度 |
| ---------------- | ---------------- | ---------- |
| Shell (1959)     | n/2, n/4, ..., 1 | O(n²)      |
| Knuth (1973)     | (3^k - 1) / 2    | O(n^(3/2)) |
| Sedgewick (1986) | 交替公式         | O(n^(4/3)) |

本實作用 **Knuth 序列**（1, 4, 13, 40, ...）。為什麼比 Shell 序列好？Shell 序列的 gap 都是偶數的倍數，導致奇偶位置的元素直到 gap=1 才互動；Knuth 序列避免了此問題。

## 複雜度

| Best       | Average    | Worst      | 空間 | 穩定 | In-place |
| ---------- | ---------- | ---------- | ---- | ---- | -------- |
| O(n log n) | 取決於 gap | O(n^(3/2)) | O(1) | ❌   | ✅       |

Knuth 序列下約 O(n^(3/2))；gap 選擇影響效能。大間距交換會破壞相等元素的相對順序 → 不穩定。
