# Counting Sort（計數排序）

準備一排標有號碼的桶子，每張牌直接丟進對應號碼的桶，再按桶序收集。Non-comparison-based，可突破 Ω(n log n) 下界。

## 觸發條件

元素為整數（負數經 `min` 偏移處理）且值域 k 不大於 O(n)。

## 適用前提

| 前提                                   | 違反時後果                         |
| -------------------------------------- | ---------------------------------- |
| 元素可映射到有限整數範圍（負數可偏移） | 無法建立計數陣列                   |
| 值域 k = O(n)                          | k >> n 時空間浪費嚴重，退化為 O(k) |

## 運作流程（執行期）

1. 找到值域 `[min, max]`，建立大小為 `max - min + 1` 的計數陣列（用 `值 - min` 當索引）。
2. 統計每個值出現的次數。
3. 對 count 做前綴和：`count[i]` 變成「值 ≤ (i + min) 的元素總數」。
4. 反向遍歷原陣列，用 `count[arr[i] - min]` 定位每個元素的最終位置。

## 逐步範例

`input = [2, 5, 3, 0, 2, 3, 0, 3]`（min = 0, max = 5, range = 6）

**Step 1 — 計數**（`count[值 - min]++`）：

| 值（索引） | 0   | 1   | 2   | 3   | 4   | 5   |
| ---------- | --- | --- | --- | --- | --- | --- |
| 出現次數   | 2   | 0   | 2   | 3   | 0   | 1   |

**Step 2 — 前綴和**（`count[i] += count[i-1]`，變成「值 ≤ i 的元素總數」）：

| 值（索引） | 0   | 1   | 2   | 3   | 4   | 5   |
| ---------- | --- | --- | --- | --- | --- | --- |
| 前綴和     | 2   | 2   | 4   | 7   | 7   | 8   |

意義：值為 `v` 的元素，最終位置區段右界是 `count[v] - 1`。例如值 3 的前綴和是 7 → 最後一個 3 放在 index 6。

**Step 3 — 反向遍歷放置**（從尾端 `arr[7]=3` 開始；放完 `count[v]--`）：

| 讀取 `arr[i]` | 用 `count[v]` | 放到 output 的 index | 放置後 `count[v]` |
| ------------- | ------------- | -------------------- | ----------------- |
| 3             | 7             | 6                    | 6                 |
| 0             | 2             | 1                    | 1                 |
| 3             | 6             | 5                    | 5                 |
| 2             | 4             | 3                    | 3                 |
| 0             | 1             | 0                    | 0                 |
| 3             | 5             | 4                    | 4                 |
| 5             | 8             | 7                    | 7                 |
| 2             | 3             | 2                    | 2                 |

`output = [0, 0, 2, 2, 3, 3, 3, 5]`。

**為什麼反向**：兩個值都是 3 的元素，原陣列中較後者先被處理、放到較右的位置（index 6），保留了原始相對順序 → 穩定。

## 關鍵洞察

**前綴和 + 反向遍歷是穩定性的關鍵**。反向遍歷確保相同值的元素保持原始相對順序；若改正向遍歷，同值的後出現元素會被放到更前面，破壞原始順序。

```javascript
count[arr[i] - min]++; // 偏移處理：保證索引 >= 0（負數也能當索引）
```

## 複雜度

| Best     | Average  | Worst    | 空間     | 穩定 | In-place |
| -------- | -------- | -------- | -------- | ---- | -------- |
| O(n + k) | O(n + k) | O(n + k) | O(n + k) | ✅   | ❌       |

k = 值域大小（max - min + 1）；k >> n 時不划算。

## 常見陷阱

- **未處理負數**：直接 `count[arr[i]]++` 會在負數索引越界。必須偏移 `count[arr[i] - min]++`。
