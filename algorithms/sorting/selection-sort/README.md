# Selection Sort（選擇排序）

每一輪從未排序區選出最小值，放到已排序區的末端。

## 觸發條件

需要最少交換次數的場景（每輪只交換一次），或資料寫入成本遠高於比較成本。

## 運作流程（執行期）

1. 外層迴圈 `i`：代表已排序區的右邊界。
2. 內層迴圈 `j`：在未排序區 `[i+1, n-1]` 中找最小值的索引。
3. 將最小值與未排序區的第一個元素交換。

## 逐步範例

`input = [5, 3, 8, 1, 2]`（`|` 左為已排序區，每輪只交換一次）

| 輪 i | 未排序區    | 找到最小值 | 交換             | 結果            |
| ---- | ----------- | ---------- | ---------------- | --------------- |
| 0    | [5,3,8,1,2] | 1 (idx 3)  | swap(0,3)        | 1 \| 3, 8, 5, 2 |
| 1    | [3,8,5,2]   | 2 (idx 4)  | swap(1,4)        | 1, 2 \| 8, 5, 3 |
| 2    | [8,5,3]     | 3 (idx 4)  | swap(2,4)        | 1, 2, 3 \| 5, 8 |
| 3    | [5,8]       | 5 (idx 3)  | minIdx===i，不換 | 1, 2, 3, 5 \| 8 |

最後一個元素自然就位 → `[1, 2, 3, 5, 8]`。全程僅 3 次交換（≤ n-1）。

## 關鍵洞察

每輪只做一次交換（總共恰好 n-1 次，O(n) swaps），但無法提前終止，best = worst = average 都是 O(n²)。

**為什麼不穩定**：例如 `[5a, 5b, 3]`，第一輪 5a 與 3 交換變成 `[3, 5b, 5a]`，5a 與 5b 的相對順序被破壞。

## 複雜度

| Best  | Average | Worst | 空間 | 穩定 | In-place |
| ----- | ------- | ----- | ---- | ---- | -------- |
| O(n²) | O(n²)   | O(n²) | O(1) | ❌   | ✅       |

交換次數最少：O(n)。

## 與 Bubble Sort 的差異

Selection Sort 每輪只交換一次但無法提前終止；Bubble Sort 交換次數可能達 O(n²)，但已排序資料可 O(n) 完成。
