# Quick Sort（快速排序）

隨便挑一張牌當基準（pivot），比它小的放左邊、大的放右邊，再對兩邊重複。分治法（Divide and Conquer）。

本目錄含兩個變體：Lomuto partition 版 `quickSort` 與三路分割版 `quickSort3Way`。

## 觸發條件

通用排序的首選（實務中常數因子最小）、不需要穩定性、有 in-place 需求。

## 運作流程（執行期）

1. **Partition**：選擇 pivot，將陣列分為「< pivot」和「≥ pivot」兩部分。
2. **Conquer**：遞迴排序左右兩部分。
3. 不需要 merge 步驟——partition 本身就把元素放到正確的相對位置（partition 後 pivot 已在最終位置，遞迴不含 pivot 本身）。

## 逐步範例（Lomuto partition）

`input = [7, 2, 1, 8, 6, 3, 5, 4]`，假設隨機化後選到 `pivot = arr[hi] = 4`。

不變量：`i` 是「< pivot 區域」的右界；`j` 掃描時遇到 `< pivot` 就 `i++` 再 swap(i, j)。

| j   | arr[j] | < 4? | 動作           | 陣列狀態（粗體 = < pivot 區） | i   |
| --- | ------ | ---- | -------------- | ----------------------------- | --- |
| 0   | 7      | ✗    | —              | 7, 2, 1, 8, 6, 3, 5, [4]      | -1  |
| 1   | 2      | ✓    | i=0, swap(0,1) | **2**, 7, 1, 8, 6, 3, 5, [4]  | 0   |
| 2   | 1      | ✓    | i=1, swap(1,2) | **2, 1**, 7, 8, 6, 3, 5, [4]  | 1   |
| 3   | 8      | ✗    | —              | **2, 1**, 7, 8, 6, 3, 5, [4]  | 1   |
| 4   | 6      | ✗    | —              | **2, 1**, 7, 8, 6, 3, 5, [4]  | 1   |
| 5   | 3      | ✓    | i=2, swap(2,5) | **2, 1, 3**, 8, 6, 7, 5, [4]  | 2   |
| 6   | 5      | ✗    | —              | **2, 1, 3**, 8, 6, 7, 5, [4]  | 2   |

掃描結束後把 pivot 換到 `i+1`：`swap(3, hi)` → `[2, 1, 3, 4, 6, 7, 5, 8]`，pivot 4 落在 index 3（最終位置）。接著遞迴排序左半 `[2,1,3]` 與右半 `[6,7,5,8]`。

## 逐步範例（三路分割 / Dutch National Flag）

`input = [3, 1, 3, 2, 3, 1]`，`pivot = arr[lo] = 3`。三區：`[lo,lt) < 3`、`[lt,gt] = 3`、`(gt,hi] > 3`。

| arr[i] | vs pivot | 動作               | lt  | i   | gt  | 陣列狀態           |
| ------ | -------- | ------------------ | --- | --- | --- | ------------------ |
| 初始   |          |                    | 0   | 1   | 5   | [3, 1, 3, 2, 3, 1] |
| 1      | <        | swap(i,lt); lt,i++ | 1   | 2   | 5   | [1, 3, 3, 2, 3, 1] |
| 3      | =        | i++                | 1   | 3   | 5   | [1, 3, 3, 2, 3, 1] |
| 2      | <        | swap(i,lt); lt,i++ | 2   | 4   | 5   | [1, 2, 3, 3, 3, 1] |
| 3      | =        | i++                | 2   | 5   | 5   | [1, 2, 3, 3, 3, 1] |
| 1      | <        | swap(i,lt); lt,i++ | 3   | 6   | 5   | [1, 2, 1, 3, 3, 3] |

`i > gt` 迴圈結束。中段 `[lt..gt] = [3..5]` 全是 pivot 3，已就最終位置，**不再參與遞迴**——只遞迴 `[lo, lt-1] = [1,2,1]` 與 `(gt, hi]`（此例為空）。這就是大量重複元素能接近 O(n) 的原因。

> **易錯點**：`arr[i] > pivot` 時 `i` 不前進——因為從 `gt` 換過來的元素還沒檢查過。

## 關鍵洞察

「重活」在 partition 步驟（與 Merge Sort 相反）。效能取決於 pivot 選擇——最差情況（每次選到最大/最小值）退化為 O(n²)，但**隨機 pivot** 或 median-of-three 可將期望複雜度保證在 O(n log n)。

## Partition 策略比較

| 策略     | 做法                              | 優點                    | 缺點                           |
| -------- | --------------------------------- | ----------------------- | ------------------------------ |
| Lomuto   | 單向掃描，splitIdx 追蹤小元素邊界 | 實作簡單                | 大量重複元素時退化為 O(n²)     |
| Hoare    | 雙向掃描，左右指標向中間逼近      | 平均交換次數少約 3 倍   | 實作較複雜，pivot 不在最終位置 |
| 三路分割 | 分成 `<`, `=`, `>` 三區           | 處理大量重複元素為 O(n) | 實作最複雜                     |

## 變體：三路分割（Dutch National Flag）

維護三個區域 `[lo, lt) < pivot`、`[lt, gt] = pivot`、`(gt, hi] > pivot`。等於 pivot 的元素不再參與後續遞迴，大量重複元素時從 O(n²) 優化到接近 O(n)。

```javascript
if (arr[i] < pivot) {
  swap(i, lt);
  lt++;
  i++;
} else if (arr[i] > pivot) {
  swap(i, gt);
  gt--;
} // i 不前進：換來的元素還沒檢查
else i++; // === pivot，留中間
```

## 複雜度

| 變體   | Best       | Average    | Worst | 空間          | 穩定 | In-place |
| ------ | ---------- | ---------- | ----- | ------------- | ---- | -------- |
| Lomuto | O(n log n) | O(n log n) | O(n²) | O(log n)~O(n) | ❌   | ✅       |
| 3-way  | O(n)       | O(n log n) | O(n²) | O(log n)~O(n) | ❌   | ✅       |

Lomuto 隨機化 pivot 避免最差；3-way 大量重複元素接近 O(n)，但用 `arr[lo]` 為 pivot，已排序資料仍 O(n²)。空間最差 O(n) 來自遞迴堆疊。

## 常見陷阱

- **未隨機化 pivot**（固定選 `arr[hi]`）→ 已排序/逆序資料退化為 O(n²)。應隨機選再交換到末端。

## 常見變體

- **小區間改用 Insertion Sort**：`if (hi - lo < CUTOFF) insertionSort(arr, lo, hi)`，減少遞迴開銷（如 Java `Arrays.sort`）。
- **Intro Sort**：Quick Sort + Heap Sort fallback（遞迴深度超過 2log n 時切換），C++ `std::sort` 的策略。
