# Insertion Sort（插入排序）

像玩牌一樣，每抽到一張新牌就插入手牌中正確的位置。

## 觸發條件

資料幾乎已排序（adaptive，best case O(n)）、小資料量（n ≤ ~50）、作為其他排序的子程式（如 Timsort、Shell Sort、Quick Sort 小區間 cutoff、[Bucket Sort](../bucket-sort/README.md) 桶內排序）。

## 運作流程（執行期）

1. 從第二個元素開始，視為「待插入的新牌」`key`。
2. 將 `key` 與前方已排序區從右到左比較。
3. 比 `key` 大的元素逐一右移（shift 而非 swap，減少賦值次數），騰出位置後插入。

## 逐步範例

`input = [5, 2, 4, 6, 1]`（`|` 左為已排序區；`key` 是當前抽出的牌）

| i   | key | 已排序區的右移過程          | 插入後結果      |
| --- | --- | --------------------------- | --------------- |
| 1   | 2   | 5 > 2，右移 5               | 2, 5 \| 4, 6, 1 |
| 2   | 4   | 5 > 4，右移 5；2 < 4 停     | 2, 4, 5 \| 6, 1 |
| 3   | 6   | 5 < 6，不右移（內層立即停） | 2, 4, 5, 6 \| 1 |
| 4   | 1   | 6,5,4,2 全 > 1，逐一右移    | 1, 2, 4, 5, 6   |

注意 `i=3` 時 key=6 已比左鄰大，內層 `while` 一次都不跑——這就是 adaptive：對幾乎有序的資料每步都是 O(1)。

## 關鍵洞察

對「幾乎有序」的資料天生高效（adaptive），因為內層迴圈在元素已就位時立即停止。這使它成為 Timsort（Python/Java 內建排序）的基礎組件。

實作上接受 `lo`/`hi` 範圍參數，方便被 Quick Sort cutoff、Bucket Sort 等場景對子區間排序。

```javascript
// 用 shift（平移）而非 swap → 賦值次數從 3n 降到 n+1；嚴格大於才移動 → 穩定
while (j >= lo && arr[j] > key) {
  arr[j + 1] = arr[j];
  j--;
}
arr[j + 1] = key;
```

## 複雜度

| Best | Average | Worst | 空間 | 穩定 | In-place |
| ---- | ------- | ----- | ---- | ---- | -------- |
| O(n) | O(n²)   | O(n²) | O(1) | ✅   | ✅       |

幾乎有序時極快，Timsort 的基礎。

## 常見變體

- **哨兵（sentinel）**：在 `arr[0]` 放 -∞ 省去 `j >= 0` 的邊界檢查（微優化）。
