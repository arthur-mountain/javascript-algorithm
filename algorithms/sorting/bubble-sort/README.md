# Bubble Sort（冒泡排序）

反覆掃過整個陣列，遇到相鄰兩個順序不對就交換，每一輪把當前最大值「冒泡」到末端。

## 觸發條件

教學用途，或資料幾乎已排序（best case O(n)，需 early termination 優化）、小資料量。

## 運作流程（執行期）

1. 外層迴圈 `i`：控制輪數（共 n-1 輪）。
2. 內層迴圈 `j`：相鄰元素兩兩比較，大的往後冒泡；每輪結束後 `arr[n-1-i]` 已是正確位置的最大值，故內層只需比到 `n-1-i`。
3. 若某一輪沒有發生任何交換 → 已排序，提前終止。

## 逐步範例

`input = [5, 3, 8, 2, 1]`（「」標示該次交換的兩個值）

**第一輪**（把最大值冒泡到末端）：

- `5 > 3` → 交換：(「3」,「5」, 8, 2, 1)
- `5 < 8` → 不動：(3, 5, 8, 2, 1)
- `8 > 2` → 交換：(3, 5,「2」,「8」, 1)
- `8 > 1` → 交換：(3, 5, 2,「1」,「8」)　← 8 已就最終位置

**第二輪**（範圍縮到 `[0, n-2]`）：

- `3 < 5` → 不動：(3, 5, 2, 1, 8)
- `5 > 2` → 交換：(3,「2」,「5」, 1, 8)
- `5 > 1` → 交換：(3, 2,「1」,「5」, 8)　← 5 就位

**第三輪**：`3 > 2`、`3 > 1` 各交換一次 → (2, 1, 3, 5, 8)；**第四輪**：`2 > 1` 交換 → (1, 2, 3, 5, 8)，本輪有交換。**第五輪**：無交換 → `swapped = false`，提前終止。

## 關鍵洞察

Bubble Sort 是唯一可透過「無交換則提前終止」優化到 best case O(n) 的簡單排序。但平均與最差情況下交換次數最多，實務中最慢。

```javascript
// 嚴格大於才交換 → 相等元素保持原始順序 → 穩定
if (arr[j] > arr[j + 1]) [arr[j], arr[j + 1]] = [arr[j + 1], arr[j]];
```

## 複雜度

| Best | Average | Worst | 空間 | 穩定 | In-place |
| ---- | ------- | ----- | ---- | ---- | -------- |
| O(n) | O(n²)   | O(n²) | O(1) | ✅   | ✅       |

Best case O(n) 需 early termination 優化。

## 常見陷阱

- 忘了 `swapped` 旗標 → 失去 best case O(n)，已排序資料仍跑滿 O(n²)。
