# Heap Sort（堆積排序）

建一個「最大堆」，反覆取出堆頂（最大值）放到末尾。

## 觸發條件

需要 O(n log n) 最差情況保證 + in-place + 不需要穩定性（不能接受 Quick Sort 的 O(n²) 最差情況）。

## 運作流程（執行期）

1. **Build Max-Heap**：將陣列原地建成 Max-Heap，從最後一個非葉節點 `floor(n/2)-1` 往上 heapify。
2. **Extract Max**：反覆將堆頂與末端交換，縮小 heap 範圍，重新 heapify。

## 逐步範例

`input = [3, 9, 2, 1, 4, 5]`（n = 6）。陣列即完全二元樹：index `i` 的子節點是 `2i+1`、`2i+2`。

```
        3(0)
       /    \
     9(1)    2(2)
    /   \    /
  1(3) 4(4) 5(5)
```

**Step 1 — Build Max-Heap**（從最後一個非葉節點 `floor(6/2)-1 = 2` 往上 heapify）：

| heapify(i) | 比較                           | 動作             | 陣列               |
| ---------- | ------------------------------ | ---------------- | ------------------ |
| i=2 (值2)  | 子節點 5 > 2                   | swap(2,5)        | [3, 9, 5, 1, 4, 2] |
| i=1 (值9)  | 子節點 1,4 都 < 9              | 不動             | [3, 9, 5, 1, 4, 2] |
| i=0 (值3)  | 子節點 9,5 → 最大 9；swap(0,1) | 下沉後子樹已滿足 | [9, 4, 5, 1, 3, 2] |

> i=0 下沉：3 與 9 換 → `[9,3,5,1,4,2]`，3 來到 index 1，再與其子 1,4 比 → 4 最大，swap → `[9,4,5,1,3,2]`。Max-Heap 完成（堆頂 9 是最大值）。

**Step 2 — Extract Max**（堆頂與末端交換，heapSize 縮 1，重新 heapify 堆頂）：

| 交換            | heapSize 縮至 | heapify 後（已排序區用 \| 分隔） |
| --------------- | ------------- | -------------------------------- |
| swap(0,5): 9↔2 | 5             | [5, 4, 2, 1, 3 \| 9]             |
| swap(0,4): 5↔3 | 4             | [4, 3, 2, 1 \| 5, 9]             |
| swap(0,3): 4↔1 | 3             | [3, 1, 2 \| 4, 5, 9]             |
| swap(0,2): 3↔2 | 2             | [2, 1 \| 3, 4, 5, 9]             |
| swap(0,1): 2↔1 | 1             | [1 \| 2, 3, 4, 5, 9]             |

完成 `[1, 2, 3, 4, 5, 9]`。

> **易錯點**：每次 heapify 的 heapSize 必須是遞減的 `i`（已排序區的左界），否則右邊已就位的最大值會被重新捲回 heap。

## 關鍵洞察

- **Build heap 是 O(n)** 而非 O(n log n)——因為大部分節點在底層，heapify 距離短。整體 O(n log n) 來自 n 次 extractMax，每次 O(log n)。
- **為什麼從下往上 build**：heapify 假設左右子樹已是 heap，由下往上才能保證此前提。
- **迭代版 heapify**：用迴圈下沉而非遞迴，省去呼叫堆疊 → 空間才真正是 O(1)。

## 複雜度

| Best       | Average    | Worst      | 空間 | 穩定 | In-place |
| ---------- | ---------- | ---------- | ---- | ---- | -------- |
| O(n log n) | O(n log n) | O(n log n) | O(1) | ❌   | ✅       |

最差情況 O(n log n) + in-place 的唯一選擇。

## 常見陷阱

- **heapify 範圍錯誤**：extract 階段必須傳遞遞減的 `i` 作為 heapSize（`heapify(arr, i, 0)`），而非固定的 `n`，否則已放到末端的最大值會被重新捲入 heap。
