# Bucket Sort（桶排序）

按數值範圍分成若干桶，桶內各自排序後串接。Non-comparison-based 的分配式排序。

> **SSOT 依賴**：桶內排序復用 [`../insertion-sort`](../insertion-sort/index.js) 的 `insertionSort`，不重複實作一份。

## 觸發條件

資料均勻分佈在某個範圍內（如 [0, 1) 的浮點數），且值域可預測。

## 運作流程（執行期）

1. 建立 k 個空桶，每個桶負責一段值域範圍。
2. 將每個元素分配到對應的桶。
3. 對每個桶內部用 Insertion Sort（桶內元素少時效率高）。
4. 按桶的順序串接所有桶的內容。

## 逐步範例

`input = [29, 25, 3, 49, 9, 37, 21, 43]`，`bucketCount = 8`（min=3, max=49）。

桶索引公式：`floor((val - min) / (max - min + 1) * bucketCount)`，此處 `= floor((val - 3) / 47 * 8)`。

**Step 2 — 分配到桶**：

| val | 計算           | 桶  |
| --- | -------------- | --- |
| 29  | floor(26/47·8) | 4   |
| 25  | floor(22/47·8) | 3   |
| 3   | floor(0/47·8)  | 0   |
| 49  | floor(46/47·8) | 7   |
| 9   | floor(6/47·8)  | 1   |
| 37  | floor(34/47·8) | 5   |
| 21  | floor(18/47·8) | 3   |
| 43  | floor(40/47·8) | 6   |

| 桶   | 0   | 1   | 2   | 3        | 4    | 5    | 6    | 7    |
| ---- | --- | --- | --- | -------- | ---- | ---- | ---- | ---- |
| 元素 | [3] | [9] |     | [25, 21] | [29] | [37] | [43] | [49] |

**Step 3 — 桶內 Insertion Sort**：只有桶 3 有兩個元素，排成 `[21, 25]`，其餘桶 ≤ 1 個免排。

**Step 4 — 按桶序串接**：`[3, 9, 21, 25, 29, 37, 43, 49]`。

**為什麼快**：資料均勻分佈時每桶平均 1~2 個元素，桶內排序近 O(1)；但若全部擠進同一桶，就退化成單一個 Insertion Sort 的 O(n²)。

> **分母為什麼是 `max - min + 1` 而非 `max - min`**：用 `+1` 確保最大值 `val === max` 時 `(val-min)/(max-min+1) < 1`，算出的桶索引落在 `[0, k-1]` 不越界（見「常見陷阱」）。代價是桶利用率略降——值域小時元素會偏向低編號桶，因此本例特意用值域較大的整數讓分佈清楚。

## 關鍵洞察

當資料均勻分佈時，每個桶平均只有 n/k 個元素。桶內 Insertion Sort 是 O((n/k)²)，k 個桶總成本 k × O((n/k)²) = O(n²/k)。取 k = n 時為 O(n)。若分佈極度不均，所有元素可能落入同一個桶，退化為 O(n²)。

## 複雜度

| Best     | Average  | Worst | 空間     | 穩定 | In-place |
| -------- | -------- | ----- | -------- | ---- | -------- |
| O(n + k) | O(n + k) | O(n²) | O(n + k) | ✅   | ❌       |

需均勻分佈；worst case 所有元素同一桶。穩定性取決於桶內排序（此處 Insertion Sort 穩定）。

## 常見陷阱

- **桶索引越界**：映射用 `(max - min)` 時，`val === max` 會算出 `bucketIdx = k`（越界）。應用 `(max - min + 1)` 作分母。
