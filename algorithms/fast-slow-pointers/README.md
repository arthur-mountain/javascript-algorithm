# Fast & Slow Pointers（快慢指標）

## 🎯 設計理念

### 存在意義

**這個演算法策略被發明出來，是為了解決什麼類型的問題？**

- **暴力法**：要偵測序列中是否有環，暴力法用 HashSet 記錄走過的節點，每到一個新節點就查是否已存在 → O(n) 時間 + O(n) 額外空間。要找鏈結串列的中間節點，暴力法先走一遍算長度再走一半 → 兩次遍歷。
- **可利用的結構特性**：在環長 C 的環內，速度 2 的指標每步比速度 1 的多走 1 步，間距以 1 遞減，最多 C 步後間距歸零——這是追趕問題（Catch-up Problem）的直接推論，保證兩指標必然相遇。在線性序列中，速度 2x 的指標走完時，速度 1x 的恰好走一半——這是速度比 2:1 的直接推論。
- **核心加速原理**：用「速度差」製造確定性的相遇/位置關係，將需要額外空間或多次遍歷的問題降至 O(1) 空間、單次遍歷。

> 注意區分兩個常被混淆的概念：
>
> - **鴿巢原理**（Pigeonhole Principle）：有限狀態空間 + 確定性函數 → 序列必然成環。回答的是「為什麼一定存在環」（如 Happy Number、287 Find Duplicate）。
> - **追趕問題**：環內速度差為 1 → 間距每步縮減 1 → 最多 C 步相遇。回答的是「為什麼不同速度的指標一定會碰上」。

### 心智模型

**類比：環形操場上跑步**

想像兩個人在環形跑道上從同一起點同時出發。一個跑得快（每秒 2 步），一個跑得慢（每秒 1 步）。無論跑道多大，快跑者遲早會「套圈」慢跑者——他們會再次相遇。這就是環偵測。

如果跑道是直線的（不是環），快跑者到達終點時，慢跑者恰好在半程——這就是找中點。

**局限**：

- 跑步類比暗示「連續移動」，但演算法是離散步進的，每步走 1 或 2 個節點
- 類比未涵蓋 Floyd 演算法的第二階段（找環入口），那需要數學證明而非直覺
- 類比只涵蓋「不同速同起點」的情境，未涵蓋「同速不同起點」（找倒數第 k 個）

## 🏗️ 運作機制

### 核心組件

| 組件                               | 功能                                            | 為什麼需要                                                                          |
| ---------------------------------- | ----------------------------------------------- | ----------------------------------------------------------------------------------- |
| 慢指標 slow                        | 每步前進 1 個節點                               | 提供基準速度；fast 到終點時 slow 恰好在中間（找中點）；在環中作為被追趕者（環偵測） |
| 快指標 fast                        | 每步前進 2 個節點（或先走 k 步後以速度 1 前進） | 速度差 = 1 保證在環中必然追上 slow；速度比 2:1 保證 slow 停在中點                   |
| 碰撞點（僅環偵測 / 環入口變體）    | fast 與 slow 相遇的位置                         | 環偵測的關鍵證據；找環入口的 Phase 2 計算起點                                       |
| 第二慢指標 pointer（僅環入口變體） | Phase 2 中從起點以速度 1 前進                   | 與碰撞點出發的 slow 同步移動，利用 `F = nC - a` 的數學性質，相遇點即環入口          |
| 距離差 k（僅找倒數第 k 個變體）    | fast 領先 slow 的固定步數                       | fast 到終點時，slow 距終點恰好 k 步 → 倒數第 k 個                                   |

### 狀態表示

- **變數與初始狀態**：

  - `slow`（慢指標位置）、`fast`（快指標位置），初始值皆為 `head`
  - 環入口變體額外需要 `pointer`，在 Phase 2 初始為 `head`
  - 找倒數第 k 個變體：`fast` 先走 k 步後才開始同步前進

- **Invariant（合法條件）**：

  - 環偵測 / 環入口：「slow 走了 t 步時，fast 走了 2t 步。兩者都在環內時，間距每步縮減 1，保證最多 C 步（C = 環長）相遇」
  - 找中點：「slow 移動的步數 = fast 移動步數的一半。因此 fast 到終點時，slow 恰好在序列中間」
  - 找倒數第 k 個：「fast 與 slow 的距離差始終為 k。因此 fast 到終點時，slow 距終點恰好 k 步」

- **Maintenance（狀態維護）**：
  每次迭代中，slow 走 1 步、fast 走 2 步（或同速 1 步）。步幅固定，invariant 被自動維護——不需要像 Sliding Window 那樣做「違反 → 修復」的操作，也不需要像 DP 那樣做狀態轉移。這是追趕型 / 等速型演算法的特性：invariant 內建在步幅規則中。

- **Extract（結果提取）**：

  - 環偵測：在每次迭代的 Advance 之後立即檢查 `slow === fast`，為 true 則提取「有環」
  - 找中點：在迴圈結束（`fast` 到終點）的瞬間，`slow` 的當前位置即為中點
  - 找環入口：Phase 1 碰撞後，Phase 2 中 `pointer === slow` 的瞬間提取環入口
  - 找倒數第 k 個：在迴圈結束（`fast === null`）的瞬間，`slow` 的位置即為答案

- **Termination（終止條件）**：
  - 有環場景：`slow === fast`（相遇 → 有環 → 結束或進入 Phase 2）
  - 無環場景 / 找中點：`fast === null || fast.next === null`（fast 到達終點）

### 適用前提條件

**使用這個策略之前，必須確認以下前提條件成立：**

| 前提條件                                           | 為什麼需要                                                                                   | 違反時的後果                                   | 替代方案                                      |
| -------------------------------------------------- | -------------------------------------------------------------------------------------------- | ---------------------------------------------- | --------------------------------------------- |
| 每個位置恰好有一個後繼（out-degree = 1）           | fast 和 slow 必須走同一條路徑才能相遇；若一個節點有多個出邊，兩者可能走不同路徑而永遠不遇    | 得到錯誤結果（false negative：有環但偵測不到） | DFS + 顏色標記（白/灰/黑）或 Topological Sort |
| 序列支持逐步前進操作（顯式或隱式的 next function） | fast/slow 需要能以不同步幅移動；若結構只支持隨機存取而無 next 概念，快慢指標無用武之地       | 無法實作指標移動                               | 直接用 index 運算（如 `array[length / 2]`）   |
| 空間約束為 O(1)（軟性前提）                        | 若允許 O(n) 空間，HashSet 更直覺且更容易實作。快慢指標的核心價值在於用 O(1) 空間達到相同效果 | 答案仍正確，但失去策略優勢                     | HashSet 記錄已訪問位置                        |

### 與資料結構的關聯

本演算法本身是一種策略（strategy），它的實作依賴底層資料結構來儲存資料與維護狀態：

| 資料結構                              | 角色     | 在本演算法中的用途                                                       | 典型題目/場景                                    |
| ------------------------------------- | -------- | ------------------------------------------------------------------------ | ------------------------------------------------ |
| Singly Linked List                    | 輸入載體 | 最典型的操作對象，節點透過 `.next` 逐步前進                              | 141 Linked List Cycle, 876 Middle of Linked List |
| Array（值作為 next 指標）             | 輸入載體 | 將 `nums[i]` 視為從 index `i` 出發的 next 指標，把陣列轉化為隱式鏈結串列 | 287 Find the Duplicate Number                    |
| 隱式序列（透過函數 `f(x)` 定義 next） | 輸入載體 | 對數字反覆施加某種運算（如各位數字平方和），產生隱式序列                 | 202 Happy Number                                 |

> 本演算法不需要「狀態維護」或「效能加速」用的輔助資料結構——它只用兩個指標變數，不依賴額外空間，這正是它相對於 HashSet 方案的核心優勢。

**關鍵洞察**：Fast & Slow Pointers 定義的是「不同速度遍歷」的抽象策略，它不在乎底層是 Linked List 還是 Array——只要能定義「下一步去哪」（next function），就能套用。辨識「隱式鏈結串列」是使用此策略的核心抽象能力。

### 流程步驟

**本演算法的流程類型**：執行期流程（Runtime Flow）

> 以下 Step 1–3 是環偵測、找環入口、找中點三個變體的共用骨架。
> 找環入口在 Step 3 後追加 Phase 2（見模式分類）。
> 找倒數第 k 個有獨立流程（Phase A + Phase B），見模式分類。

---

1. **Initialize（初始化指標）**

**目的**：設定 slow 和 fast 的起始位置。

**關鍵決策點**：兩個指標是否從同一位置出發？

- 環偵測 / 找中點 / 找環入口：`slow = head, fast = head`（同起點）

```javascript
let slow = head;
let fast = head;
```

---

2. **Advance（以不同步幅前進）**

**目的**：每次迭代中，slow 走 1 步、fast 走 2 步，利用速度差製造位置關係。

**關鍵決策點**：迴圈條件如何設定？

- `while (fast !== null && fast.next !== null)` — 確保 fast 能安全走兩步
- 找中點偏左版本改為 `while (fast.next !== null && fast.next.next !== null)` — 讓 slow 少走一步

```javascript
while (fast !== null && fast.next !== null) {
  slow = slow.next; // slow 走 1 步
  fast = fast.next.next; // fast 走 2 步
  // 顯式串列：slow = slow.next
  // 隱式序列（Array）：slow = nums[slow]
  // 隱式序列（函數）：slow = f(slow)
}
```

---

3. **Extract（提取結果）**

**目的**：根據 fast 和 slow 的最終狀態，提取答案。此步驟的具體行為由變體決定。

**關鍵決策點**：

- 環偵測：在迴圈內部 Advance 後檢查 `slow === fast`，為 true 即有環
- 找中點：迴圈結束時 `slow` 即為中點，直接回傳
- 找環入口：碰撞後需進入 Phase 2（見模式分類的變體 B）

```javascript
// 環偵測：在 Advance 內部提取
if (slow === fast) return true;

// 找中點：在迴圈結束後提取
return slow;
```

### 模式分類

**變體總覽**：

```plaintext
Fast & Slow Pointers
├── 環偵測（Cycle Detection）：判斷序列是否有環
├── 找環入口（Cycle Entry Point）：定位環開始的位置（Floyd 完整演算法，Phase 1 + Phase 2）
├── 找中點 — 偏右（Find Middle — Right-biased）：偶數長度時回傳後半段第一個
├── 找中點 — 偏左（Find Middle — Left-biased）：偶數長度時回傳前半段最後一個
└── 找倒數第 k 個（Kth from End）：同速不同起點，利用固定距離差定位（獨立流程）
```

---

**變體 A：環偵測（Cycle Detection）**

**觸發條件**：題目問「是否存在環」、序列是否會無限循環、數字序列是否會重複。

**與基礎版本的差異**：這就是基礎版本。Phase 1 only——只需判斷 fast 和 slow 是否相遇。

```javascript
/**
 * 環偵測（Cycle Detection）- 判斷序列是否包含環
 *
 * @param {ListNode} head - 鏈結串列的頭節點
 * @return {boolean} 是否存在環
 *
 * 適用場景：判斷 Linked List 是否有環、數字序列是否會循環
 * 觸發條件：需要 O(1) 空間偵測環，或無法使用 HashSet
 *
 * 時間複雜度：O(n) - fast 最多走 n 步到終點（無環）或在環中最多 C ≤ n 步追上 slow（有環）
 * 空間複雜度：O(1)
 */
function hasCycle(head) {
  // 前提：head 可為 null（空序列 = 無環）；每個節點 out-degree = 1
  // Step 1: Initialize — 兩指標從同一起點出發
  let slow = head;
  let fast = head;

  // Step 2: Advance — fast 能安全走兩步時持續前進
  // 複雜度來源：fast 每次走 2 步，最多 n/2 次迭代 → O(n)
  while (fast !== null && fast.next !== null) {
    slow = slow.next; // slow 走 1 步
    fast = fast.next.next; // fast 走 2 步
    // 隱式序列適配：slow = nums[slow] / slow = f(slow)
    // 隱式序列適配：fast = nums[nums[fast]] / fast = f(f(fast))

    // Step 3: Extract — 在 Advance 後立即檢查
    // 速度差 = 1，環內間距每步縮減 1 → 最多 C 步相遇
    if (slow === fast) return true;
  }

  // fast 到達終點 → 序列有限 → 無環
  return false;
}
```

**變體 B：找環入口（Cycle Entry Point / Floyd's Complete Algorithm）**

**觸發條件**：題目問「環從哪裡開始」、找重複元素（陣列轉隱式鏈結串列）。

**與環偵測的差異**：在 Phase 1（環偵測）確認有環後，增加 Phase 2——從起點和碰撞點同時出發以速度 1 前進，相遇點即環入口。Phase 2 利用了 `F = nC - a` 的數學性質：

> 設 F = 起點到環入口的距離，a = 環入口到碰撞點，C = 環長。
> 碰撞時 slow 走了 `F + a` 步，fast 走了 `F + a + nC` 步。
> 因為 fast 速度 = 2 × slow：`2(F + a) = F + a + nC` → `F = nC - a`。
> 從碰撞點再走 `F` 步 = 在環中走 `nC - a` 步 = 回到環入口。

```javascript
/**
 * 找環入口（Floyd's Complete Algorithm）- 找到環開始的節點
 *
 * @param {ListNode} head - 鏈結串列的頭節點
 * @return {ListNode|null} 環入口節點，若無環回傳 null
 *
 * 適用場景：找鏈結串列環的起點、陣列中的重複數字（287）
 * 觸發條件：不只要知道有環，還要知道環從哪開始
 *
 * 時間複雜度：O(n) - Phase 1 最多 O(n)，Phase 2 最多 O(n)
 * 空間複雜度：O(1)
 */
function detectCycleEntry(head) {
  // 前提：head 可為 null；每個節點 out-degree = 1
  let slow = head;
  let fast = head;

  // Phase 1: 環偵測 — 找碰撞點（與變體 A 完全相同）
  // 複雜度來源：同變體 A，O(n)
  while (fast !== null && fast.next !== null) {
    slow = slow.next;
    fast = fast.next.next;

    if (slow === fast) {
      // Phase 2: 找環入口（與變體 A 的差異：追加此階段）
      // 從起點和碰撞點同時以速度 1 前進，相遇點即環入口
      let pointer = head;
      // 易錯點：必須用新變數 pointer 從 head 出發，不可直接複用 fast
      while (pointer !== slow) {
        pointer = pointer.next;
        slow = slow.next;
      }
      return pointer;
    }
  }

  return null;
}
```

**變體 C：找中點 — 偏右（Find Middle — Right-biased）**

**觸發條件**：需要找鏈結串列的中間節點，偶數長度時回傳後半段第一個。如 876 Middle of the Linked List。

**與環偵測的差異**：不需要碰撞檢查。迴圈條件為 `while (fast !== null && fast.next !== null)`，迴圈結束時 slow 直接就是答案。

- 奇數長度 `[1,2,3,4,5]`：slow 停在 3（正中間）
- 偶數長度 `[1,2,3,4]`：slow 停在 3（偏右中點，後半段第一個）

```javascript
/**
 * 找中點 — 偏右（Find Middle — Right-biased）
 *
 * @param {ListNode} head - 鏈結串列的頭節點
 * @return {ListNode} 中間節點（偶數長度時回傳後半段第一個）
 *
 * 適用場景：直接回傳中間節點、876 題
 * 觸發條件：需要單次遍歷找到中點，不知道總長度，偶數長度時要後半段起點
 *
 * 時間複雜度：O(n) - fast 走完整個串列（n/2 次迭代），slow 走一半
 * 空間複雜度：O(1)
 */
function findMiddle(head) {
  // 前提：head !== null（至少一個節點）
  // Step 1: Initialize
  let slow = head;
  let fast = head;

  // Step 2: Advance — fast 到終點時 slow 在中間
  // 複雜度來源：fast 每次走 2 步，遍歷 n 個節點 → n/2 次迭代 → O(n)
  while (fast !== null && fast.next !== null) {
    slow = slow.next;
    fast = fast.next.next;
  }

  // Step 3: Extract — 迴圈結束的瞬間，slow 即為偏右中點
  return slow;
}
```

**變體 D：找中點 — 偏左（Find Middle — Left-biased）**

**觸發條件**：需要找鏈結串列的中間節點，偶數長度時回傳前半段最後一個。如 Merge Sort 分割需要在中點前斷開。

**與偏右版本（變體 C）的差異**：迴圈條件改為 `while (fast.next !== null && fast.next.next !== null)`，讓 slow 在偶數長度時少走一步。

- 奇數長度 `[1,2,3,4,5]`：slow 停在 3（正中間，與偏右相同）
- 偶數長度 `[1,2,3,4]`：slow 停在 2（偏左中點，前半段最後一個）

```javascript
/**
 * 找中點 — 偏左（Find Middle — Left-biased）
 *
 * @param {ListNode} head - 鏈結串列的頭節點
 * @return {ListNode} 中間節點（偶數長度時回傳前半段最後一個）
 *
 * 適用場景：Merge Sort 分割鏈結串列（需要在中點前斷開）
 * 觸發條件：偶數長度時需要前半段的尾節點
 *
 * 時間複雜度：O(n)
 * 空間複雜度：O(1)
 */
function findMiddleLeft(head) {
  // 前提：head !== null（至少一個節點）
  // 易錯點：若串列只有 1 個節點，fast.next 為 null → 條件不成立 → 直接回傳 head，正確
  let slow = head;
  let fast = head;

  // 與偏右版本（變體 C）的差異：檢查 fast.next.next 而非 fast.next
  // 這讓 slow 在偶數長度時少走一步，停在前半段最後一個
  while (fast.next !== null && fast.next.next !== null) {
    slow = slow.next;
    fast = fast.next.next;
  }

  // Step 3: Extract — 迴圈結束的瞬間，slow 即為偏左中點
  return slow;
}
```

**變體 E：找倒數第 k 個（Kth from End）**

**觸發條件**：題目要求找（或刪除）鏈結串列的倒數第 k 個節點。

**與其他變體的差異**：嚴格來說這是「同速不同起點」而非「不同速同起點」。核心思想相同——利用指標間的固定距離差提取位置資訊——但流程不同：

**獨立流程**：

- Phase A（Stagger）：fast 先走 k 步，建立距離差
- Phase B（Lockstep）：兩指標以相同速度前進，直到 fast 到終點

```javascript
/**
 * 找倒數第 k 個（Kth from End）- 利用固定距離差定位
 *
 * @param {ListNode} head - 鏈結串列的頭節點
 * @param {number} k - 從末尾數第 k 個（1-indexed）
 * @return {ListNode} 倒數第 k 個節點
 *
 * 適用場景：找倒數第 k 個節點、刪除倒數第 k 個節點（19 題）
 * 觸發條件：單向鏈結串列無法從尾部回推，需要一次遍歷定位
 *
 * 時間複雜度：O(n) - Phase A 走 k 步 + Phase B 走 n-k 步 = n 步
 * 空間複雜度：O(1)
 */
function findKthFromEnd(head, k) {
  // 前提：head !== null；k ≤ 鏈結串列長度（否則 fast 會在 Phase A 中越界）
  let slow = head;
  let fast = head;

  // Phase A: Stagger — fast 先走 k 步，建立距離差
  // 易錯點：k 可能等於鏈結串列長度，此時 fast = null，slow = head 就是答案
  for (let i = 0; i < k; i++) {
    fast = fast.next;
  }

  // Phase B: Lockstep — 同速前進，保持距離差為 k
  // 與變體 A-D 的差異：兩指標速度相同（都走 1 步），非不同速
  while (fast !== null) {
    slow = slow.next;
    fast = fast.next;
  }

  // fast 到終點時，slow 距終點恰好 k 步 → 倒數第 k 個
  return slow;
}
```

### 複雜度總結

| 變體          | 時間複雜度 | 空間複雜度 | 備註                                       |
| ------------- | ---------- | ---------- | ------------------------------------------ |
| 環偵測        | O(n)       | O(1)       | fast 在環中最多繞一圈（C 步）追上 slow     |
| 找環入口      | O(n)       | O(1)       | Phase 1 + Phase 2 各 O(n)，合計仍 O(n)     |
| 找中點 — 偏右 | O(n)       | O(1)       | fast 走完全程（n/2 次迭代），slow 走一半   |
| 找中點 — 偏左 | O(n)       | O(1)       | 同偏右版本，只差迴圈條件                   |
| 找倒數第 k 個 | O(n)       | O(1)       | Phase A 走 k 步 + Phase B 走 n−k 步 = n 步 |

**符號定義**：

- `n`：序列中的節點總數（含環中的節點）
- `C`：環的長度（僅環偵測/環入口適用），C ≤ n

> 注意：所有變體的時間複雜度皆為 O(n)，而非 O(n/2) 或 O(log n)。雖然 slow 只走了 n/2 步，但 fast 走了 n 步，迭代次數仍為 O(n) 級別。此策略的價值不在於降低時間複雜度，而在於將空間從 O(n)（HashSet）降至 O(1)，或將遍歷次數從 2 次降至 1 次。

### 識別核心抽象

> 這個工具的核心對象是：**序列中的位置（position in a sequence）**
>
> 它管理的是這些對象之間的什麼關係？**兩個位置之間的「速度差」或「距離差」，以及由此產生的確定性相遇/位置關係**

### 映射範例

**題目 A**：287. Find the Duplicate Number

| 維度           | 具體問題                         | 抽象映射                                       |
| -------------- | -------------------------------- | ---------------------------------------------- |
| ① 序列識別     | 陣列 `nums`，`next(i) = nums[i]` | 隱式鏈結串列，index → value 是 next 指標       |
| ② 目標類型     | 找重複數字 = 找環入口            | 變體 B：Floyd's Complete Algorithm             |
| ③ 速度配置     | 2:1 同起點 + Phase 2             | 標準 Floyd                                     |
| ④ 隱式 vs 顯式 | 隱式（陣列值作為指標）           | `slow = nums[slow]`, `fast = nums[nums[fast]]` |

**從映射到實作的關鍵步驟**：

1. 認出「n+1 個元素、值域 [1,n]」→ 鴿巢原理 → 必有重複 → 必有環
2. 定義 next function：`next(i) = nums[i]`
3. 起點是 index 0（因為值域 [1,n]，沒有值指向 0，所以 0 一定在環外）
4. 套用 Floyd 變體 B，環入口就是重複的數字

**題目 B**：202. Happy Number

| 維度           | 具體問題                             | 抽象映射                                                               |
| -------------- | ------------------------------------ | ---------------------------------------------------------------------- |
| ① 序列識別     | 數字序列，`next(n) = 各位數字平方和` | 隱式序列，數學運算定義 next                                            |
| ② 目標類型     | 判斷是否最終到達 1                   | 變體 A：環偵測，但額外檢查碰撞值是否為 1                               |
| ③ 速度配置     | 2:1 同起點                           | 標準環偵測                                                             |
| ④ 隱式 vs 顯式 | 隱式（數學函數）                     | `slow = sumOfSquares(slow)`, `fast = sumOfSquares(sumOfSquares(fast))` |

**從映射到實作的關鍵步驟**：

1. 認出「反覆施加運算」→ 有限狀態空間（數字位數有上限）+ 確定性函數 → 鴿巢原理 → 序列必然進入環
2. Happy Number 的環在 1（1 → 1 → 1...），非 Happy Number 的環不經過 1
3. 用環偵測找到 `slow === fast`，檢查是否為 1

## 🔍 觸發器(模式識別)

### 正向觸發器

看到什麼特徵應該想到這個演算法？

| 層級     | 特徵                                               | 為什麼這個特徵指向這個演算法                                                                                                                                                                    |
| -------- | -------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 關鍵字   | cycle、circular、loop、repeated、happy number      | 直接暗示「環」的存在                                                                                                                                                                            |
| 關鍵字   | middle、midpoint、half                             | 直接暗示找中點                                                                                                                                                                                  |
| 結構     | Singly Linked List（無法回頭）                     | 鏈結串列無 random access，快慢指標是 O(1) 空間找位置的唯一方式                                                                                                                                  |
| 結構     | 陣列值域 = index 範圍（如 n+1 個元素，值 ∈ [1,n]） | 值可作為 next 指標 → 隱式鏈結串列 → Floyd                                                                                                                                                       |
| 目標     | 偵測是否「回到已見過的狀態」且需要 O(1) 空間       | HashSet 需要 O(n)，快慢指標是 O(1) 替代方案                                                                                                                                                     |
| 操作模式 | 反覆施加確定性函數（f(f(f(x)))...）                | 有限狀態空間 + 確定性 → 鴿巢原理 → 必然有環 → Floyd 可用                                                                                                                                        |
| 約束     | 「不得修改原陣列」+「O(1) 額外空間」               | 排除 HashSet（O(n) 空間）和排序法（修改陣列），快慢指標是少數能同時滿足的策略。注意：此策略不改變時間複雜度（仍為 O(n)），其價值在於空間從 O(n) 降至 O(1)。因此約束觸發點是空間限制，非時間限制 |

**一句話觸發規則**：看到「序列 + 環偵測 + O(1) 空間」或「鏈結串列 + 找中間/倒數位置」→ 想到 Fast & Slow Pointers。

### 反向觸發器

什麼情況下「看起來適用但其實不是」？

| 陷阱情境                              | 為什麼不適用                                                                                   | 更好的選擇                                   |
| ------------------------------------- | ---------------------------------------------------------------------------------------------- | -------------------------------------------- |
| 有向圖中的環偵測（非鏈結串列）        | 每個節點可能有多個 next（out-degree > 1），fast/slow 可能走不同路徑而永遠不遇 → false negative | DFS + 顏色標記（白/灰/黑）/ Topological Sort |
| 陣列中找重複但不需要 O(1) 空間        | 快慢指標需要「值域 = index 範圍」才能建立隱式鏈結串列；若空間無限制，HashSet 更簡單            | HashSet O(n) 空間直接記錄                    |
| 找鏈結串列的任意第 k 個節點（非倒數） | 正向第 k 個直接走 k 步即可，不需要兩個指標                                                     | 線性遍歷 k 步                                |
| Doubly Linked List 找中點             | 雙向串列可以從兩端同時走，不需要速度差                                                         | 雙指標從兩端向中間收斂                       |
| 偵測「值的重複」而非「位置的環」      | 快慢指標偵測的是結構上的環（位置循環），不是值相等                                             | Sorting + 相鄰比較 / HashSet                 |

### 與類似工具的決策點

```plaintext
序列中偵測環
├── 每個節點只有一個 next（out-degree = 1）
│   ├── 需要 O(1) 空間 → Fast & Slow Pointers
│   └── 允許 O(n) 空間 → HashSet（更直覺）
└── 有向圖（out-degree > 1）→ DFS + visited/in-stack 狀態

鏈結串列找特定位置
├── 找中點 → Fast & Slow Pointers（單次遍歷）
│   ├── 偶數長度要後半段起點 → 偏右版本（變體 C）
│   └── 偶數長度要前半段結尾 → 偏左版本（變體 D）
├── 找第 k 個（正向）→ 直接遍歷 k 步
├── 找倒數第 k 個 → Fast & Slow Pointers（變體 E）/ 先算長度再走 n-k 步
└── 需要來回移動 → 考慮轉成 Array 或用 Doubly Linked List

陣列找重複數字
├── 可用額外空間 → HashSet
├── 可修改陣列 → 原地標記（nums[abs(x)] *= -1）
└── 不可修改 + O(1) 空間 + 值域 = [1,n] → Floyd's Algorithm（隱式鏈結串列，變體 B）
```

## ⚠️ 注意事項

### 常見陷阱

| 陷阱                            | 錯誤寫法                                                                | 正確寫法                                                                | 為什麼                                                                                        |
| ------------------------------- | ----------------------------------------------------------------------- | ----------------------------------------------------------------------- | --------------------------------------------------------------------------------------------- |
| fast 未做 null 檢查就存取 .next | `while (fast.next)`                                                     | `while (fast !== null && fast.next !== null)`                           | 偶數長度時 `fast.next.next` 使 fast 變為 null，下一輪直接存取 `null.next` 會 crash            |
| 找中點的時間複雜度誤判          | `// Time: O(log n)  ← 誤以為「走一半」就是 log`                         | `// Time: O(n)  ← fast 走了 n 步，slow 走 n/2，總步數仍線性`            | 省的是遍歷次數（2 pass → 1 pass），不是漸進複雜度。O(n/2) = O(n)                              |
| 找環入口 Phase 2 忘記重設指標   | `// Phase 2 直接用 fast 繼續` `while (fast !== slow) { ... }`           | `let pointer = head;` `while (pointer !== slow) { ... }`                | Phase 2 需要一個指標從 head 出發（距環入口 F 步），與 slow（距環入口 nC-a 步）同速前進        |
| 隱式鏈結串列起點選錯            | `let slow = nums[0]; let fast = nums[0];` `// ← 跳過了 index 0`         | `let slow = 0; let fast = 0;` `// ← 從 index 0 出發`                    | 值域 [1,n]，沒有值指向 index 0，所以 0 保證在環外。從 nums[0] 開始等於跳過了起點              |
| 偶數長度中點偏左偏右搞混        | `while (fast !== null && fast.next !== null)` `// ← 偏右，但以為是偏左` | `while (fast.next !== null && fast.next.next !== null)` `// ← 明確偏左` | 偏右讓 slow 多走一步停在後半段首個；偏左讓 slow 少走一步停在前半段末個。Merge Sort 分割需偏左 |

### 常見變體

| 變體             | 修改內容                                                                                      | 適用場景                                                         |
| ---------------- | --------------------------------------------------------------------------------------------- | ---------------------------------------------------------------- |
| 步幅調整         | 在任意變體模板上，將 fast 步幅從 2 改為 k                                                     | 找 1/k 位置點（罕見，但原理相同）                                |
| 找環長度         | 在環偵測模板（變體 A）的碰撞點處，固定 slow 讓 fast 繼續走一圈計數                            | 需要知道環的長度                                                 |
| 隱式序列 + Floyd | 在環偵測（變體 A）或找環入口（變體 B）模板上，將 `slow.next` 替換為 `nums[slow]` 或 `f(slow)` | Happy Number（函數定義 next）、Find Duplicate（陣列值定義 next） |
